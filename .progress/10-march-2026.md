## Step 2вЂ“4 Fix: Torch Version Conflict

**Status:** Complete

The initial SLURM install job (22470128) completed Steps 2вЂ“4, but `lightning==2.2.2` in `requirements.txt` pulled in `torch==2.4.1` as a transitive dependency, overwriting `torch==2.0.1+cu118`. This broke `torch-cluster` (ABI mismatch).

A fix job (22470405) on `gpu-a100-short` resolved it:

```bash
pip install torch==2.0.1 torchvision==0.15.2 torchaudio==2.0.2 \
    --index-url https://download.pytorch.org/whl/cu118 --force-reinstall --no-deps
pip install triton==2.0.0
pip install --force-reinstall torch-cluster -f https://data.pyg.org/whl/torch-2.0.1+cu118.html
pip install "huggingface_hub<0.24"
```

Additionally rebuilt pytorch3d and chamferdist against the correct torch. All imports verified on GPU node (A100 80GB) and login node.

## Step 5e: Breaking Bad Dataset (everyday subset)

**Status:** Complete

The raw mesh data from the [Breaking Bad Dataset](https://breaking-bad-dataset.github.io/) is needed for visualization/rendering and optionally for generating point clouds from scratch.

### Downloads from Dataverse

Downloaded `everyday_compressed.zip` (377 MB) and `data_split.tar.gz` (25 KB) from [Borealis Dataverse](https://doi.org/10.5683/SP3/LZNPKB):

```bash
wget -O everyday_compressed.zip "https://borealisdata.ca/api/access/datafile/341985"
wget -O data_split.tar.gz "https://borealisdata.ca/api/access/datafile/342180"
```

### Decompression Environment

Cloned the Breaking-Bad repo and created a separate conda env for decompression:

```bash
git clone https://github.com/Breaking-Bad-Dataset/Breaking-Bad-Dataset.github.io.git
conda create -n breaking-bad python=3.8 -y
conda activate breaking-bad
conda install numpy scipy tqdm -y
conda install -c conda-forge igl -y
pip install gpytoolbox==0.2.0
```

### Decompression

Extracted compressed data to `/data/gpfs/projects/punim2657/bb_data/` and submitted decompression as SLURM batch jobs on `sapphire` partition:

```bash
unzip everyday_compressed.zip -d bb_data/
tar xzf data_split.tar.gz -C bb_data/
python decompress.py --data_root /data/gpfs/projects/punim2657/bb_data --subset everyday --category all
```

- **Initial job (22472862):** Processed 17/20 categories before hitting disk quota (~444 GB total project usage due to Rabati2025 at 336 GB).
- **Fix:** Migrated Rabati2025 (336 GB) to MediaFlux, freeing space (see `MEDIAFLUX_MIGRATION_Rabati2025.md`).
- **Resume job (22584618):** Re-ran for Bowl, Vase, WineGlass. All completed successfully.
- **Final output:** 50 GB of decompressed mesh `.obj` files across 20 categories.
- **Note:** 4 meshes (1 Bowl, 3 Vase) have empty source data in the upstream dataset вЂ” not a decompression error.

### Organization

Moved decompressed data into PuzzleFusion++ expected structure:

```bash
BB_REPO=/data/gpfs/projects/punim2657/Breaking-Bad-Dataset.github.io
BB_DATA=/data/gpfs/projects/punim2657/bb_data
mkdir -p $BB_REPO/data/breaking_bad
mv $BB_DATA/everyday $BB_REPO/data/breaking_bad/
cp $BB_DATA/data_split/everyday.train.txt $BB_REPO/data/breaking_bad/
cp $BB_DATA/data_split/everyday.val.txt $BB_REPO/data/breaking_bad/
```
