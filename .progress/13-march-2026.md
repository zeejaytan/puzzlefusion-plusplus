### SLURM Script

Created `inference_test.slurm` as a smoke test (100 val samples, full 6-iteration auto-agglomerative pipeline):

```bash
python test.py \
    experiment_name=everyday_epoch2000_bs64 \
    denoiser.data.val_batch_size=1 \
    denoiser.data.data_val_dir=./data/pc_data/everyday/val/ \
    denoiser.data.matching_data_path=./data/matching_data/ \
    denoiser.data.overfit=100 \
    denoiser.ckpt_path=output/denoiser/everyday_epoch2000_bs64/training/last.ckpt \
    verifier.ckpt_path=output/verifier/everyday_epoch100_bs64/training/last.ckpt \
    inference_dir=test_run \
    verifier.max_iters=6
```

`overfit=100` limits the val set to the first 100 samples for a quick end-to-end check.

## Step 8: Result Visualisation

**Status:** Complete

Visualised the inference output for **sample #97** (Bottle, 13 fragments, part accuracy = **84.6%**), selected as the highest-accuracy multi-fragment sample in the test run.

### Scripts

- `export_viz_data.py` — exports per-fragment point clouds and the 120-step denoising trajectory from the inference output to `viz_data.json` (0.8 MB)
- `assembly_comparison.png` — before / predicted / GT comparison image
- `assembly_trajectory.png` — denoising convergence image (13 keyframes + error curve)
- `assembly_viz.html` — self-contained interactive three.js viewer (orbit controls, animation player, per-fragment status panel)

### Visualisation approach

`part_pcs_gt` from the val `.npz` file (canonical assembled point clouds, 1000 pts/fragment) was used as the base geometry. For each fragment, the relative pose error between `pred[-1]` and `gt` was computed and applied to show predicted vs. ground truth positions. Trajectory keyframes were interpolated from the 120 denoising steps.

### assembly_comparison.png

Three-panel view:

- **Left — Scattered (before):** 13 fragments placed in a grid, each in a random orientation — the model input
- **Middle — Model output:** Predicted assembly. Red ✗ markers flag 4 misassembled fragments (F2, F3, F4, F6, rotation error > 30°). The remaining 9 are correctly placed
- **Right — Ground truth:** Canonical assembly from the dataset

### assembly_trajectory.png

13 keyframe views across the 120 denoising steps (6 auto-aggl iterations × 20 DDPM steps each), with panel titles colour-coded red → green as mean rotation error decreases. Bottom panel shows the error convergence curve:

- **Step 0:** 116.5° mean rotation error (noisy, scattered)
- **Step 120:** 44.9° mean rotation error (mostly assembled)
