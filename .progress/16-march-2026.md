## 16 March 2026 — PuzzleFusion++ viewer fix

- **Goal**: Get the interactive assembly viewer working again as a simple, download-and-open HTML file after mesh-toggle experiments broke the previous version.
- **Issue**: `assembly_viz.html` was still using the newer async `main()` pattern and mesh-toggle hooks, so opening it (especially from local disk on a laptop) threw a "Failed to initialise viewer" alert.
- **Change**: Deleted the old `assembly_viz.html` (which still contained mesh/animation code and error handling) and recreated it from scratch as a minimal, standalone viewer:
  - Inlines the data from `viz_data.json` into a `const DATA = {...}` literal (single Breaking Bad bottle sample).
  - Renders each fragment as a colored point cloud in three.js using `base_points` plus per-fragment rigid transforms (`scattered_tf`, `pred_tf`, `gt_tf`).
  - Provides three modes only: Scattered, Predicted, Ground truth (no meshes, no animation timeline, no external fetch).
  - Uses only ES module imports from `https://esm.sh/three@0.158.0` and `OrbitControls`, with no dependency on any local HTTP server.
- **Result**: `assembly_viz.html` is once again a self-contained, point-cloud-only viewer; it can be copied to a laptop and opened directly in a browser tab (no server, no extra assets) to inspect the assembled bottle before/after.
